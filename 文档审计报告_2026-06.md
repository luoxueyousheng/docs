## P0 — 跨文档高危（照抄即失败 / 进程崩溃 / API 语义反了）

### P0-1　`app-ready` 成功判定写错，照抄代码窗口永远建不出来
几乎所有教程都写 `if (window_id == 1 && strcmp(event_data, "success") == 0)`。源码成功时 `event_data` 是 JSON `{"ok":true,"message":"success"}`（`src/app/event_loop.rs:51`），`strcmp` 永不为 0 → **永远走 else，窗口不创建**。失败时才是纯文本 `"<code>: <message>"`（`src/ffi/c_api/lifecycle.rs:86`）。
- 正确判据：只看 `window_id`（成功 `1` / 失败 `0`，见头文件 `include/JadeView.h:481-484`），或解析 JSON 取 `ok`。
- 受影响：`spec/quickstart.md`(行83/271/337/398/474/511)、`spec/custom-titlebar.md`(行30/118/288)、`spec/ipc-communication.md`(行69/165)、`spec/local-web-resources.md`(行48)、`api/event-types.md`、`api/index.md`。
- 额外问题：`api/event-types.md` 与 `api/index.md` 对该载荷的描述**互相矛盾**，且**都**与源码不符（event-types 说成功是纯文本"success"、index 说失败是 JSON，正好反了）。

### P0-2　`WebViewSettings.disable_right_click` 字段名错且语义相反
源码字段名是 **`allow_right_click`**（`0=禁用, 1=允许`，NULL 默认=允许），不是 `disable_right_click`（`include/JadeView.h:26`、`src/core/types.rs:64`、`src/window_manager/window_creation.rs:622`）。文档按"1=禁用"使用会得到**相反**结果，零初始化结构体（值 0）实际会**禁用**右键。
- 受影响：`api/window-api.md`(行73)、`api/window-styling.md`(行124/139/162，三处)、`api/index.md`(行186)、`spec/quickstart.md`(行115，照抄无法编译)。

### P0-3　`set_protocol_service_path` 文档示例少传第 4 参 `hot_reload`
源码是 **4 参** `set_protocol_service_path(root_path, url_buffer, buffer_size, hot_reload)`（`include/JadeView.h:722`、`src/ffi/c_api/system.rs:204-209`）。函数是 `__stdcall`，少传一个参数 → 栈不平衡 / 崩溃。
- 受影响：`api/japk.md`(行78-82)、`api/japk-load-memory.md`(C 行216 + Python argtypes 行273-277)、`spec/quickstart.md`(行189-193)、`spec/local-web-resources.md`(行59-63/261)、`spec/distribution.md`(行93)、`api/local-server-api.md`(新旧对照表 行44)。

### P0-4　结构体字段漏写 → 固定 ABI 静默错位
文档强调结构体字段顺序必须与库一致，但字段表漏了源码中真实存在的字段，少填会导致**后续字段整体错位**（库读到错误内存），且不报错：
- `WebViewSettings.cors_whitelist` —— 在 `postmessage_whitelist` 与 `autofill` 之间（`include/JadeView.h:44-46`、`src/core/types.rs:74`）。`api/index.md`(字段表 行183-196)漏列。（注：`api/window-api.md` 已列，OK。）
- `WebViewWindowOptions.skip_taskbar` / `no_activate` —— 结构体末尾两字段（`include/JadeView.h:176-182`、`src/core/types.rs:144,146`）。`api/index.md`(字段表 行151-170)漏列。

### P0-5　`spec/custom-titlebar.md` 事件回调签名错误（ABI 不匹配）
示例把回调写成 `int app_ready_callback(uint32_t, const char* event_type, const char* event_data)`（3 参、返回 int，行29/117/286-287）。真实 `IpcCallback` 是 **`const char*(uint32_t, const char*)`**（2 参、返回指针，`include/JadeView.h:426`）。注册即未定义行为。

### P0-6　`spec/local-web-resources.md` 自定义协议 URL 整篇写错
文档全篇用 `https://{appname}.local/`（行11/14/67/95/236/262/274/277），源码实际是 **`JADE://{app_signature 小写}`**，且明确**只用 `app_signature`、不用 `app_name`**（`src/ffi/c_api/system.rs:219-230`，注释原话"使用 app_signature…不使用 app_name"）。
